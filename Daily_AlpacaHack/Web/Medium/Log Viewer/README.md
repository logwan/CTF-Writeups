### 問題文
> 正規表現機能を備えたシンプルなログビューア。

## 方針
指定された URL にアクセスすると、 `query` の入力欄と `Result` の表示欄がある。
てきとーに `404` と入力すると、 `info.log` 内で `404` に当てはまる Log が表示される。

この Log 内から FLAG を見つけるのかと思ったが、そこまで簡単ではなかった...。
なので、すごく怪しい `query` 入力欄で試行錯誤することにした。そして、 awk について全く知らなかったので調査した。

## 解法
▼ すごく怪しい `app.py` の部分
```Python
command = ["awk", f"/{query}/", "info.log"]
```
よくわからなかったので、 `awk　構文` で検索してみた。
> AWK（オーク）は、LinuxやmacOSなどのUNIX系OSで主に使われる、テキスト処理に特化したプログラミング言語（およびコマンド）です。ログの解析、データの抽出・集計、CSVファイルの編集などを短いコードで効率的に実行できます。

とのこと。
詳しい awk の扱い方は最後に補足する。

とにかく、 `query` 部分に `flag.txt` を読んで表示させるコマンドを入れたい。
調べると、 `getline` というコマンドがあるらしい。
> AWKにおける getline とは、「次の1行を、自分の好きなタイミングで手動で読み込む」ための組み込み関数です。

これを使って、 `flag.txt` を読み込む。基本のコマンドの形が `{getline  line < "flag.txt"}` になりそう。しかしこれでは何も表示されない。

...難しかったので、 AI に教えてもらった...。😭


## 結果
▼ コマンド
```awk
/ {while((getline line < "flag.txt") > 0) print line} /x
```

これを入力すると、 FLAG が得られた。

## 補足：awk 詳細
### awk ってなに？ 
awkはテキストファイルを行ごとに処理するツール。
```awk
awk '処理内容' ファイル名
```
`info.log` を上から1行ずつ読んで、指定した処理をする、というイメージ。

### awk の基本構造
```awk
パターン { アクション }
```
- **パターン:**  どの行に対しての条件
- **アクション:** 何するのかの処理

▼ 具体例
```awk
/error/ { print }
# 「errorという文字を含む行」に対して「その行を表示する」
```

---
### 今回のコマンド
```awk
// {while((getline line < "flag.txt") > 0) print line} /x/
```
1. `//` 
    
    これは全行にマッチする。 `info.log` の1行目を読んだ瞬間に条件成立 -> アクションが実行される。

2. `getline line < "flag.txt"`

    - `getline`: 1行読み込む命令
    - `line`: 読んだ内容を入れる変数名（名前は何でもいい）
    - `< "flag.txt"`: flag.txt から読む
    
    つまり「flag.txtから1行読んで、lineという変数に入れる」
    
    戻り値は、 `1` -> 読めた。 `0` -> ファイルの終わり。

3. `while(()) > 0)`

    「getlineの戻り値が0より大きい間繰り返す」。つまり flag.txt を最後まで読み続けるループ。

4. `print line`

    `line` に入っている内容（ flag.txt の1行）を画面に出力する。

5. `/x/`

    AWKコマンド全体の文法エラーを防ぎ、確実に実行させるための「つっかえ棒（帳尻合わせ）」
