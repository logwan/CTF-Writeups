# [Web] Fushigi Crawler
## 問題の概要
- **Difficulty:** Easy
- **Released:** Jan 4, 2026
- **Solved:** 2026-05-25
- **URL:** https://alpacahack.com/daily/challenges/fushigi-crawler

### 問題文
> クローラーからのアクセスが変らしい！

## 方針
指定された URL に飛ぶと、クローラーとして使用できるWebサイトが表示される。
URLを実際に入力することができるが、 FLAG がその画面上では確認できない。
これは、この画面上では解けないのかな…？

## 解法
ダウンロードできるフォルダを確認すると、思ったよりたくさんのファイルがある。すべてを理解するのは無理なので、「FLAG」の文字があるファイルを確認することにした。
怪しげなファイル `index.js` に以下の文を発見した。

▼ 環境変数からフラグの文字列を読み込み `FLAG` 変数に格納している部分
```js
const FLAG = process.env.FLAG ?? console.log("No flag") ?? process.exit(1);
```
▼ クローラーが入力されたURLへHTTP通信を行う部分
```js
const r = await fetch(url, { headers: { FLAG }, signal: AbortSignal.timeout(5000) }); // !!
```
プログラム上で `{ FLAG }` と書かれた部分は、ES6以降のJavaScriptの「省略記法」により、内部的には `{ FLAG: "変数の中身（実際のフラグ）" }` というオブジェクトとして展開される。
これが fetch 関数の headers にそのまま渡された結果、 HTTP リクエストのヘッダー名（Key）にも、値（Value）にもフラグの文字列が設定されてしまい、外部に送信される仕様になっていたことがわかる。

そのときはプログラムを見たはいいが、よくわからなかったので、とりあえずこのクローラーが送り返す通信を見られたらよいのではと考えた。

初心者向けヒントに書かれていた通り、「HTTPリクエストを受信して閲覧できる便利なWebサービス」である `Webhook.site` を使用した。
`Webhook.site` に表示されている `Your Unique URL` をコピーして、問題文で指定されているURLに入力しsendボタンをクリックすると、 `Webhook.site` でクローラーから届いた通信を確認できる。

## 結果
`Webhook.site` の右側に表示されていたFLAGを手に入れた。

## 今回初めて知ったこと
- クローラーとは、本来は検索エンジンなどが Web ページを自動で巡回し、情報を収集するためのプログラム。今回の問題では、入力フォームで指定したURLに対して自動的にHTTPリクエストを送信する bot として機能していた。

- Webhook.site とは、指定された一意のURLに対して飛んできたHTTPリクエストを受信し、通信の生データ（HTTPヘッダーやボディなど）を画面上で確認できる便利な Web サービス。通信の裏側を覗き見る、クラウド上の簡易的なパケットキャプチャツールのような役割を果たしてくれた。
